# Política de seguridad

Practikalia es un proyecto en fase temprana (sin versiones publicadas todavía), así que no hay una matriz de versiones con soporte de seguridad: cualquier corrección se aplica sobre la rama principal.

## Reportar una vulnerabilidad

Si encuentras una vulnerabilidad, repórtala **en privado**, nunca en un issue o PR público.

Vía preferida: la pestaña **Security → Report a vulnerability** de este repositorio en GitHub (aviso de seguridad privado).

Incluye, si es posible:

- Descripción del problema y su impacto.
- Pasos para reproducirlo.
- Versión, rama o commit afectado.

Se revisará el aviso y se mantendrá informado su progreso hasta la resolución.

## Alcance

Esta política cubre el código de este repositorio. No cubre instancias desplegadas por terceros: cada centro que autohospede Practikalia es responsable de la seguridad de su propio despliegue (credenciales, `JWT_SECRET`, actualizaciones, etc.).
